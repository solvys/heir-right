Hermes runtime

Two modes. One agentic layer.

Don't confuse them — Local Hermes runs on your machine. Cloud Hermes is the hosted runtime for teams and sync.

Local Hermes

Your machine. Your config. Your port.

You set up Hermes on your computer. Fintheon reads your Hermes config, discovers the port automatically, and routes all local agentic actions through it.

  • Detects port from ~/.hermes/config.json
  • Health-check fallback across common ports
  • Private — data never leaves your machine for local runs
Cloud Hermes

Shared runtime for teams.

When you need collaboration, scheduled runs, or a Hermes instance without local setup, Cloud Hermes handles agentic actions in Fintheon's hosted layer.

  • Authenticated via your Fintheon account
  • Team workspaces and shared agent streams
  • Fallback when Local Hermes is unavailable

FAQ

Common questions.

Do I need Hermes installed before using Fintheon?

For Local Hermes actions, yes — you need Hermes configured on your machine. Fintheon will detect the port automatically. Cloud Hermes works without local setup for team and hosted workflows.

What's the difference between Local Hermes and Cloud Hermes?

Local Hermes runs on your computer and handles private, on-device agentic actions. Cloud Hermes is Fintheon's hosted runtime for shared team workflows and sync. The app routes each action to the correct mode — they are not interchangeable.

How do I get the macOS app?

After sign-up, your onboarding screen offers a DMG download from our latest GitHub Release. Install it, open Fintheon, and it connects to Local Hermes automatically.

How do updates work?

Fintheon uses electron-updater with GitHub Releases. When a new signed build is published, the app notifies you and installs updates in the background.
