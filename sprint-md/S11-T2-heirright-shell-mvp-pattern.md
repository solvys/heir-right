# Sprint Brief: S11-T2 -- HeirRight Shell MVP Pattern

## Goal

Use the HeirRight MVP shell as the first proven pattern before extracting a reusable Solvys shell.

## Build Scope

- HeirRight-specific nav candidates: Source Runs, Dossiers, Lead Reports, CRM Queue, Documents, Blockers, Settings.
- CRM/work-queue home screen candidate.
- Chat-first command center variant note.
- Bottom composer integration.
- Agent activity drawer.
- Local dry-run artifact preview.
- Settings surface for lead-quality toggles, including source-signal weighting, thresholds, reason codes, and review-only guardrails.

## Exclusions

- Extracting the generic shell before MVP validation.
- Treating HeirRight nav as the universal default.

## Acceptance

- HeirRight shell feels probate/sales-specific.
- Settings let operators refine lead-quality criteria without approving live outreach or changing the public client roadmap.
- MVP validates whether CRM-first or chat-first home screen is better.
- Extraction notes identify which parts are reusable and which are HeirRight-only.
